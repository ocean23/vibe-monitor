Electron 二进制缺失导致 pnpm dev 报错

📅 2026-06-01 macOS pnpm Electron 42

问题现象

在新机器上执行 pnpm dev 时，构建阶段正常完成，但随后报错并退出：

vite v7.3.3 building ssr environment for development...
✓ built in 83ms
electron main process built successfully
...
error during start dev server and electron app:
Error: Electron uninstall
    at getElectronPath (file:///.../electron-vite/dist/chunks/lib-q6ns0vZr.js:155:19)
    at startElectron (...)
[ELIFECYCLE] Command failed with exit code 1.
关键错误信息 Error: Electron uninstall — electron-vite 找不到 Electron 可执行文件，读取 path.txt 失败。

根本原因

Electron 的 npm 包本身不包含二进制文件，需由 install.js 从 GitHub Releases 下载对应平台的压缩包并解压。 安装成功后会生成两个产物：

产物路径作用
path.txt node_modules/.pnpm/electron@x.x.x/node_modules/electron/path.txt 记录可执行文件相对路径，electron-vite 读取此文件定位二进制
dist/ …/electron/dist/Electron.app（macOS） 实际的 Electron 应用程序包

关键变化：从 Electron 42 起，官方移除了下载二进制的 postinstall 脚本（出于供应链安全——postinstall 是 npm 投毒的常见入口）。 install.js 仍随包发布（挂为 install-electron 这个 bin），但 npm/pnpm install 不再自动执行它，改为「首次运行 electron 的 bin 时按需懒下载」。 而 electron-vite 的 getElectronPath 是直接读 path.txt、读不到就抛错，走不到触发懒下载那一步，于是二进制缺失的问题在 pnpm dev 时才暴露。

本项目已自动处理 根 package.json 的 postinstall 已显式调用 node node_modules/electron/install.js 下载二进制，正常一条 pnpm install 即可装好。下面的手动步骤仅在你用了 --ignore-scripts、或下载因网络失败时才需要。

快速诊断

执行以下命令确认 Electron 二进制是否存在：

# 检查 path.txt
cat node_modules/.pnpm/electron@*/node_modules/electron/path.txt

# 检查 dist 目录（macOS）
ls node_modules/.pnpm/electron@*/node_modules/electron/dist/Electron.app

若两条命令均报错或无输出，即为本文描述的问题。

解决方案

通过国内镜像手动执行 Electron 的安装脚本（先确认实际版本号）：

# 1. 确认 Electron 版本
cat node_modules/.pnpm/electron@*/node_modules/electron/package.json | grep '"version"'
# 输出示例：  "version": "42.3.0"

# 2. 用国内镜像重新下载（替换版本号）
ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/" \
  node node_modules/.pnpm/electron@42.3.0/node_modules/electron/install.js

# 3. 验证
cat node_modules/.pnpm/electron@42.3.0/node_modules/electron/path.txt
# 预期输出：Electron.app/Contents/MacOS/Electron
验证成功后 直接运行 pnpm dev，无需重新 install。

永久配置（国内网络推荐）

把镜像写进全局 ~/.npmrc，之后每次 pnpm install 触发的二进制下载都会走国内镜像：

echo 'electron_mirror=https://npmmirror.com/mirrors/electron/' >> ~/.npmrc
不要写进项目的 .npmrc 并提交 本项目是公开仓库，硬编码国内镜像会拖慢海外用户。镜像只配在你本机的全局 ~/.npmrc。

常用 Electron 镜像地址

镜像地址备注
npmmirror（推荐） https://npmmirror.com/mirrors/electron/ 国内速度最快，同步及时
GitHub Releases（官方） https://github.com/electron/electron/releases/download/ 需能访问 GitHub

总结

  1. Electron 42 起官方移除了下载二进制的 postinstall；本项目改由根 postinstall 调 install.js 安装，正常 pnpm install 即可装好。
  2. 报错信息 Error: Electron uninstall 即代表 path.txt 不存在（二进制没装上）。
  3. 若被 --ignore-scripts 跳过或下载失败：用 ELECTRON_MIRROR 指定国内镜像，手动执行 install.js 即可修复。
  4. 把镜像固化到全局 ~/.npmrc（勿提交进项目），避免新成员重复踩坑。
